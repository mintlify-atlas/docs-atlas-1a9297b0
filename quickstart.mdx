## Get Started in Minutes

Color Palettes work automatically in Dune Analytics. Simply query protocols by their names or aliases, and colors are applied to your visualizations.

<Steps>
  <Step title="Write Your Query">
    Use protocol names or aliases in your query results. Color Palettes will automatically detect them.
  </Step>
  <Step title="Create Visualization">
    Generate a chart from your query results. Colors are applied automatically.
  </Step>
  <Step title="Customize (Optional)">
    Override colors manually or use the "Restore colors" button to revert to standard palettes.
  </Step>
</Steps>

## Automatic Color Application

Color Palettes automatically apply when protocol or chain names match any defined alias (case-insensitive).

### Basic Example: Chain Analysis

<CodeGroup>

```sql Query
SELECT
  blockchain,
  COUNT(*) as transaction_count
FROM ethereum.transactions
WHERE block_time >= NOW() - INTERVAL '7' day
GROUP BY blockchain
```

```json Color Mapping
// Automatically applied colors:
{
  "ethereum": "#88AAF1",  // Matches alias "ethereum"
  "base": "#0052FF",       // Matches alias "base"
  "optimism": "#FF0420",   // Matches alias "optimism"
  "arbitrum": "#12AAFF"   // Matches alias "arbitrum"
}
```

</CodeGroup>

<Tip>
  The query column values (`ethereum`, `base`, etc.) are automatically matched against the Color Palettes aliases. No extra configuration needed!
</Tip>

### DeFi Protocol Example

Analyzing DEX volumes across protocols:

<CodeGroup>

```sql Query
SELECT
  project as protocol,
  SUM(amount_usd) as volume_24h
FROM dex.trades
WHERE block_time >= NOW() - INTERVAL '1' day
GROUP BY project
ORDER BY volume_24h DESC
LIMIT 10
```

```json Color Mapping
// Automatically applied when project names match aliases:
{
  "uniswap": "#F50DB4",     // Uniswap
  "1inch": "#0077FF",       // 1inch
  "Pancakeswap": "#F3BA2F", // Matches BNB alias "bsc"
  "sushiswap": "#...",      // If defined in palette
  "curve": "#..."           // If defined in palette
}
```

</CodeGroup>

## Understanding Alias Matching

The matching system is case-insensitive and flexible:

<Accordion title="What Matches">

**Using Ethereum as example** (`"aliases": ["eth", "weth", "ethereum"]`):

<Check>**These WILL match:**</Check>

- `eth` (exact match)
- `ETH` (case insensitive)
- `Eth` (case insensitive)
- `weth` (alias match)
- `WeTh` (case insensitive alias)
- `ethereum` (alias match)
- `evm_<color:eth>` (special syntax support)

<Warning>**These will NOT match:**</Warning>

- `ether` (not an alias)
- `evm<color:eth>` (missing underscore)
- `color:eth` (incomplete syntax)
- `ethereum_mainnet` (extra text)

</Accordion>

### Special Color Syntax

For advanced use cases, you can use the special `<color:alias>` syntax:

```sql
SELECT
  CONCAT('Layer2_', '<color:', blockchain, '>') as chain_label,
  COUNT(*) as tx_count
FROM transactions
GROUP BY blockchain
```

This allows you to append custom text while still getting automatic color matching.

## Customization Options

### Override Colors Manually

You can override any automatically applied color:

1. Click on the color in your visualization settings
2. Choose your preferred color
3. The override persists for that specific chart

### Restore Default Colors

To revert to standard Color Palettes:

1. Open your visualization settings
2. Click the **"Restore colors"** button
3. All colors revert to the community-maintained standard

<Info>
  **Best Practice**: Use standard colors for consistency, but override when you need specific branding or accessibility requirements.
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Colors not applying automatically">
    **Check these common issues:**
    
    1. **Alias mismatch**: Your query result might use a name variation not in the aliases list
    2. **Case sensitivity**: While matching is case-insensitive, ensure no extra characters
    3. **Protocol not in palette**: The protocol might not be added yet - consider contributing!
    
    **Solution**: Check the `color-palettes.json` file for exact alias matches, or override colors manually.
  </Accordion>

  <Accordion title="Want to use a protocol not in the palette">
    **You have two options:**
    
    1. **Manual override**: Set custom colors for that specific chart
    2. **Contribute**: Add the protocol to Color Palettes so everyone benefits
    
    Contributing helps the entire Dune community maintain consistent visualizations!
  </Accordion>

  <Accordion title="Need different colors for branding">
    **Override colors manually:**
    
    1. Click the color in your visualization settings
    2. Select your brand color
    3. The chart will use your custom color
    
    You can always use "Restore colors" to revert to standard palettes later.
  </Accordion>
</AccordionGroup>
